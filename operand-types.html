Operand Types

Operand arguments can easily be manipulated using the following prefixes and suffixes.

  • ""

    • Explanation: Double quotes ("") represent a string. A string is used to enclose multiple characters.
    Example: PNT "This sentence must be enclosed in quotes in order to work correctly."

  • RX

    • Explanation: Any operand beginning with 'R' tells the processor to resolve a value from a specified register; Where X is a number between (and including) 0 - 15.
    Example:
    INC R0
    INC R1
    ADD, R2, R1, R0

  • Xd

    • Explanation: (Where X is a number) Any number, followed by a lowercase 'd' is considered a decimal literal.
    Example:
    INC R0
    INC R0
    ADD R1, R0, 5d
    PNT R1
    Output:
    7

  • Xb

    • Explanation: (Where X is a combination of 0's and 1's) Any number containing 0's and 1's only, followed by a lowercase 'b', is considered a binary literal.

  • Xh

    • Explanation: (Where X is a hexadecimal number) Any number, followed by a lowercase 'h' is interpreted as a hexadecimal number.
