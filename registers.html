Special Registers

A number of BASM's registers have been reserved for uses in future development and temporary data storage. These "special registers" are detailed below.

  • Register 15

    • Register fifteen should not be used by used to store data in programs, as this register is written to when an error arises in the program. The value contained in R15 changes to '1' when a fault occurs.

  • Register 14

    • Register fourteen is reserved for future use by the Virtual Machine's processor. Although it is possible for earlier versions to use this register, it is recommended you do not use it in order to promote compatibility and good practice.

  • Register 13

    • This register is used for storing temporary data. It is possible to use this register, however be warned that the processor may overwrite this register in order to store obsolete junk.
