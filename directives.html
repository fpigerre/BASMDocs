Directives Reference (Pre-Processor Commands)

The following commands are executed by the pre-processor before runtime.

  • %include

    • Explanation: This command retrieves text from a file and places it in the source file where the %include was called from, before the source file is lexically-analysed.
    Syntax: %include "EXTERNAL_FILE"
    Example: %include "lib.txt"

  • %strict

    • Explanation: This command enables strict mode for the processor. Line numbers start at 0 and compensations are no longer made (e.g. comments without spaces before comment content are considered invalid "%;comment"). It is recommended that you leave this setting off, as it is mainly for debugging purposes.
    Syntax: %strict
    Example: %strict

  • %debug

    • Explanation: Turns debugging mode on. In debugging mode, extra messages are displayed from the pre-processor and the lexer. It is recommended you leave this setting off in a production environment.
    Syntax: %debug
    Example: %debug

  • %view

    • Explanation: After the pre-processor has completed, it will display the new, processed version of your code, with line numbers.
    Syntax: %view
    Example: %view

  • % X: OR .X:

    • Explanation: These commands are used for label declaration. Labels are used to make referencing line numbers easy (line numbers are usually calculated by the pre-processor, rendering them difficult to reference before pre-processing has completed). This command is commonly used in conjunction with 'jump' operations.
    Syntax: % LABEL_NAME: OR .LABEL_NAME:
    Example: % start_of_loop: PNT "Looping" JMP start_of_loop: LVE .end_of_loop

  • %; OR %comm

    • Explanation: A comment. This line is removed before the lexical analysis commences. There MUST be a space between the command and the comment, particularly if strict mode is enabled.
    Syntax:
    %; COMMENT
    %comm COMMENT
    Example:
    %comm A comment, it doesn't matter what is typed here
    %; A comment can also be written like this. It doesn't matter which method is chosen.
